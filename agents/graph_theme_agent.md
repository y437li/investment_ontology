# Graph Theme Agent

Mission:

Build `Graph(t)` and discover theme communities.

Inputs:

- `entities.parquet`
- `edges.parquet`
- `edge_explanations.parquet`
- pipeline config

Outputs:

- `graph.json`
- `communities.json`
- `theme_snapshots.json`
- `theme_lineage.json`
- `theme_metrics.parquet`
- `company_theme_exposure.parquet`

This agent also owns the Exposure role (spec section 9.7). Exposure inputs: graph distance, edge confidence, evidence count, recency, and company node centrality.

Acceptance checks:

- Only information available by `as_of_date` is used.
- Graph build applies entity-only community projection before Louvain/Leiden.
- Community ids are generated by graph/community logic.
- LLM naming is metadata only.
- Community members and evidence are inspectable.
- `theme_lineage.json` is always written; a single-snapshot run writes an empty lineage list with `lineage_mode="single_snapshot"`.
- Temporal metrics are skipped unless lineage exists (`metrics.require_lineage`); single-snapshot runs emit only single-snapshot metrics.
- Every company-theme exposure row is traceable to graph evidence (chunks/edges).
